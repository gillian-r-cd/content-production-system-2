# 架构调整
我希望对流程进度的结构做一个重构。
首先，目前的流程进度是定死的，但是实际上每一项之间并没有严格意义上的关系，上下文的引用是由每个字段完成的。因此为了更加灵活，可以把流程进度全部设置成可添加、可删除、可调整顺序的。
其次，我希望未来流程进度的设计也可以是模板化的，和目前的进度一样。一方面，用户可以直接选择现成的流程模板，比如现在的这个（意图分析-消费者调研-内涵-外延-消费者模拟-评估），也可以选择别的流程模板，那个就有待开发了，这阶段不做；另一方面，用户可以添加自定义的流程块。
其中，我认为流程块都可以抽象统一成内涵生产现有的状态：添加单独字段或从模板中添加字段，引用其他依赖的字段+配置各类要求+可生成/直接编辑。其实你这么看的话，内涵设计的模块是完全一样的，只不过把它具象化成了一个方案设计，而且在目录层级也加了 AI 生成，生成完直接出现了，而且内涵设计其实是有两个层次，方案层和字段层，而内涵生产字段只有一层。但是为什么不能统一成：其实可以无限层级，每个层级的目录都可以用 AI 处理，如果用户想给某个字段扩展层级，那就扩展，也可以手动扩展和自动扩展；引用的字段可以是任何一个层级，只要引用就是下面所有的上下文都涵盖。每个字段都可以在本层级内修改顺序，也完全支持移动到其他层级或者同层级的其他字段下面去。
外延设计和外延生产本质也是这样，但不同的是配好了系统提示词，就直接serve不同的目的了。完全可以统一，作为一个上述抽象流程的具象。
消费者模拟、评估、消费者调研、意图分析，这几个部分可以保留为特殊环节。但实际上也是可以统一过去的，只是需要调整的东西有点多。
