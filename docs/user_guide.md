## 1. 系统启动

### 1.1 配置 API 密钥

首次使用前，你需要配置 OpenAI API 密钥：

```bash
cd backend
cp env_example.txt .env
```

编辑 `.env` 文件，填入你的 API Key：

```
OPENAI_API_KEY=sk-your-api-key-here
OPENAI_MODEL=gpt-5.1
```

### 1.2 启动服务

**启动后端**（新开一个终端）：

```bash
cd backend
.\venv\Scripts\activate   # Windows
python main.py
```

**启动前端**（新开一个终端）：

```bash
cd frontend
npm run dev
```

### 1.3 访问系统

- 🖥️ **工作台**: http://localhost:3000/workspace
- ⚙️ **后台设置**: http://localhost:3000/settings
- 📖 **API 文档**: http://localhost:8000/docs

## 2. 后台设置（首次使用必做）

在创建项目之前，建议先完成后台基础设置。访问 http://localhost:3000/settings

### 2.1 创建「创作者特质」

创作者特质是所有项目的基础上下文，描述你/你的团队/你的品牌的核心特征。

1. 点击「创作者特质」标签
2. 点击「新建」
3. 填写信息：
   - **名称**：例如「产品设计团队」
   - **描述**：团队的专业背景、擅长领域
   - **声音调性**：写作风格，如「专业但亲和」「轻松幽默」
   - **核心价值观**：团队坚持的理念

**示例**：
```
名称：效率工具产品组
描述：专注于帮助知识工作者提高效率的产品团队，有5年B2B SaaS经验
声音调性：专业、简洁、注重实用价值
核心价值观：效率至上、用户第一、持续迭代
```

### 2.2 创建「字段模板」（可选）

字段模板是可复用的内容结构。如果你经常生产同类内容，建议预设模板。

1. 点击「字段模板」标签
2. 点击「新建」
3. 填写：
   - **模板名称**：例如「产品介绍模板」
   - **字段列表**：添加多个字段
     - 字段名称：如「产品定位」「核心功能」「用户收益」
     - AI 提示词：指导 AI 如何生成这个字段
     - 生成前提问：生成前需要用户回答的问题

### 2.3 配置「模拟器」

模拟器用于模拟不同类型的用户体验。系统预置了几种类型：

| 类型 | 适用场景 | 说明 |
|------|---------|------|
| **阅读式** | 文章、课程 | 模拟用户阅读全文后的反馈 |
| **对话式** | Chatbot、客服 | 模拟多轮对话交互 |
| **决策式** | 销售页、落地页 | 模拟购买决策过程 |
| **探索式** | 帮助文档、FAQ | 模拟用户寻找答案 |
| **体验式** | 产品功能 | 模拟完成特定任务 |

你可以新建或修改模拟器，调整评估维度。

### 2.4 配置「渠道」（用于外延生产）

渠道定义了你的内容要发布到哪里。常见渠道：

- 小红书（图文+标签）
- 公众号（长文）
- 抖音（短视频脚本）
- 销售 PPT
- 官网 Landing Page

每个渠道可以设置字数限制、格式要求、内容风格等。

## 3. 创建第一个项目

### 3.1 进入工作台

访问 http://localhost:3000/workspace

### 3.2 新建项目

1. 在左上角找到项目选择器
2. 点击「新建项目」
3. 填写：
   - **项目名称**：例如「效率工具 Pro 产品发布」
   - **选择创作者特质**：选择你刚才创建的
   - **是否使用 DeepResearch**：建议勾选，获得更深入的用户洞察

4. 点击「创建」

### 3.3 理解工作台布局

工作台分为三栏：

```
┌──────────────┬──────────────────────┬──────────────────┐
│   左栏        │       中栏           │      右栏         │

│ ○ 消费者调研 │                      │                  │
│ ○ 内涵设计   │                      │  你：@产品定位    │
```

## 4. 内容生产流程详解

### 4.1 意图分析（第一步）

当你创建项目后，Agent 会自动开始意图分析。

**Agent 会问你三个关键问题**，例如：

1. 「这个内容的核心目标是什么？是获客、教育用户还是品牌建设？」
2. 「目标受众是谁？他们目前面临什么问题？」
3. 「你希望用户看完后采取什么行动？」

**你的回答非常重要**，会作为后续所有内容生成的基础上下文。

> 💡 **提示**：回答尽量详细具体，不要只说「提高转化率」，而是说「让中小企业主在看完后愿意预约产品演示」。

### 4.2 消费者调研（DeepResearch）

如果你启用了 DeepResearch，系统会：

1. 🔍 搜索相关市场信息
2. 📊 分析目标用户画像
3. 📝 输出：
   - **总体画像**：目标用户群体特征
   - **核心痛点**：他们面临的问题
   - **价值点**：你的内容能提供什么价值
   - **典型人物小传**：3-5个具体的用户故事

**人物小传示例**：

> 张明，35岁，某互联网公司产品总监。每天处理大量需求评审和项目跟踪，经常加班到深夜。尝试过多种项目管理工具，但觉得都太复杂。希望找到一个「上手即用」的效率工具，不需要培训团队就能用起来。

这些人物小传可以直接用于后续的消费者模拟。

### 4.3 内涵设计 → 内涵生产

**内涵设计**：Agent 会根据意图和调研，建议内容大纲和字段结构。

你可以：
- 接受 Agent 的建议
- 修改大纲结构
- 选择使用已有的字段模板
- 设置字段之间的依赖关系

**内涵生产**：按照大纲逐字段生成内容。

每个字段生成前，你可以：
- 查看/修改该字段的 AI 提示词
- 回答生成前提问
- 选择依赖哪些已完成的字段

### 4.4 外延设计 → 外延生产

**外延设计**：选择要发布的渠道（小红书、公众号、PPT等）

**外延生产**：根据渠道要求，将核心内容改编为适合该渠道的版本。

例如，同一个产品介绍：
- 小红书版：短小精悍 + emoji + 标签
- 公众号版：深度解读 + 案例故事
- PPT 版：结构化要点 + 视觉建议

### 4.5 调整阶段顺序

左栏中，「内涵设计」「内涵生产」「外延设计」「外延生产」这四个阶段**可以拖拽调整顺序**。

常见调整场景：
- 先做外延（快速测试市场反应），再做内涵
- 跳过外延，只做核心内容
- 先设计所有，再统一生产

## 5. 与 Agent 对话

右栏是你与 AI Agent 交流的地方。Agent 不只是回答问题，它能真正**执行任务**。

### 5.1 使用 @ 引用

在对话中输入 `@` 会弹出可引用的内容列表：

```
你：@产品定位 这个表述太技术化了，能改得更接地气一点吗？

AI：好的，我来调整一下产品定位的表述...
    [修改后的内容]
    
    需要我直接更新这个字段吗？
```

可引用的内容：
- `@字段名` - 引用已生成的任何字段
- `@消费者调研` - 引用调研结果
- `@张明` - 引用特定的人物小传

### 5.2 常用对话意图

| 你说 | Agent 会做 |
|------|-----------|
| 「@产品定位 太长了，缩短到100字」 | 修改指定字段 |
| 「帮我对比一下小红书和公众号的版本」 | 分析比较 |
| 「这个产品的竞品有哪些？」 | 可能触发 DeepResearch |
| 「用张明的视角重新写一下」 | 切换人物视角重写 |
| 「一键生成剩下的所有字段」 | 批量生成（如果依赖满足） |

### 5.3 对话历史功能

- **编辑重发**：点击你发送的消息，可以编辑后重新发送
- **重试**：点击 Agent 回复旁的「重试」，让 Agent 重新生成
- **复制**：每条消息都可以一键复制

## 6. 消费者模拟

当核心内容生成完成后，进入「消费者模拟」阶段。

### 6.1 创建模拟

1. 点击左栏「消费者模拟」
2. 点击「新建模拟」
3. 选择：
   - **模拟器类型**：阅读式/对话式/决策式等
   - **目标内容**：选择要测试的字段
   - **模拟人物**：从调研中选择，或自定义

### 6.2 查看模拟结果

模拟完成后，你会看到：

**对话式模拟示例**：
```
[消费者] 张明：这个工具能导入我现有的任务吗？
[服务方]：可以的，我们支持从 Excel、Notion 等导入...
[消费者] 张明：那团队协作怎么用？
[服务方]：您可以邀请团队成员...
[消费者] 张明：好的，我了解了。

评估反馈：
- 响应相关性：8/10 - 回答切题
- 问题解决率：7/10 - 基本解答了主要问题
- 交互体验：8/10 - 流畅自然

未解答的问题：定价方案
摩擦点：没有主动介绍核心优势
```

### 6.3 利用模拟改进内容

模拟结果会直接反馈到评估报告中。如果发现：
- 某个问题反复被问到 → 内容中需要补充说明
- 用户有顾虑没被打消 → 增加信任要素
- 某部分理解困难 → 需要简化表达

## 7. 评估与优化

### 7.1 生成评估报告

在「评估」阶段，系统会综合分析：
- 所有已生成的内容
- 消费者模拟的反馈
- 与意图的匹配度

输出一份结构化的评估报告，包含各维度评分和具体建议。

### 7.2 采纳建议

评估报告中的每条建议，你可以：
- ✅ **采纳**：Agent 会根据建议自动修改内容
- ❌ **忽略**：跳过这条建议
- ✏️ **修改后采纳**：调整建议内容后执行

### 7.3 迭代优化

采纳建议后，建议重新运行消费者模拟，验证改进效果。

## 8. 进阶功能

### 8.1 版本管理

当你需要大幅修改已有内容时，建议创建新版本：

1. 点击项目名称旁的「...」菜单
2. 选择「创建新版本」
3. 输入版本说明，如「v2: 调整目标受众为企业用户」

新版本会复制所有现有字段，你可以在此基础上修改而不影响原版。

### 8.2 设置 Agent 自主权

在左栏底部，点击「Agent 自主权设置」：

- **全自动**：Agent 完成所有阶段，你只看最终报告
- **每阶段确认**：Agent 完成一个阶段后等你确认再继续
- **每字段确认**：每生成一个字段都需要你确认

### 8.3 查看调试日志

在后台设置 → 调试日志，你可以看到：
- 每次 AI 调用的完整输入/输出
- Token 使用量和成本
- 生成耗时

支持导出为 JSON/CSV，便于分析优化。

### 8.4 快捷操作

| 快捷操作 | 说明 |
|---------|------|
| 一键生成 | 按依赖顺序批量生成所有待生成字段 |
| 导出项目 | 导出所有字段内容为 Markdown/JSON |
| 复制项目 | 基于现有项目创建新项目 |

## 常见问题

### Q: 生成的内容质量不理想怎么办？

1. 检查意图分析是否足够具体
2. 调整字段的 AI 提示词
3. 尝试不同的人物视角
4. 在对话中提供更多上下文后重新生成

### Q: 字段之间的依赖关系有什么用？

依赖关系确保内容的连贯性。例如：
- 「产品卖点」依赖「用户痛点」，这样卖点会针对痛点来写
- 「FAQ」依赖「产品功能」，确保问答与功能描述一致

### Q: 模拟结果和真实用户差距大吗？

模拟是基于 AI 对典型用户的理解。建议：
- 使用调研中生成的详细人物小传
- 多用不同类型的人物模拟
- 将模拟结果作为参考，结合真实用户测试
