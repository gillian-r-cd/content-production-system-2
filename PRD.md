## 系统描述及结构说明

这是一套对所有商业内容生产通用的平台，Agent 引领整个内容生产的思路推进和微调。

这套系统通过下面的结构进行组织：

- 全局要素
    - 系统提示词，包括：消费者调研、内涵生产、外延生产、意图分析
    - 字段模板：可新建，具体怎么建见下面的说明
    - Simulator 管理：每个不同的内容项目，就需要不同的 Simulator 结构，比如需要交互的，比如只是一次性 review 的，比如需要回答结构性问题的，等等）
    - 渠道管理：用于外延设计，可以设置一些关键的、常用的渠道，比如销售PPT、小红书等
    - Agent 管理
- 项目变量
    - 创作者特质
    - 项目意图
    - 字段关联关系等，见下方说明

## 内容生产流程

内容生产全流程分为几个步骤，由 Agent 引领：

1. 意图分析：由 Agent 提出三个探寻内容生产意图的最相关最有效的问题。用户的回答需要是后面上下文的一部份分。
2. 消费者调研：调研以上意图的消费者 / 客户并输出总体画像、痛点、价值点、多个典型的消费者人物小传（完整的故事背景）。消费者人物小传也是一个字段，可编辑可选，选中的人物小传直接传导到[消费者模拟]。通过 DeepResearch 的 Agentic workflow 实现，用 LangChain 实现。
3. 内容生产（以下每个步骤都可以调整顺序）
    - 内涵设计：根据意图分析、消费者调研，建议内容生产方案和大纲（及引用的字段模板）
    - 内涵生产：根据内涵设计，用定好的生产方案和大纲及字段模板生产具体的内容
    - 外延设计：根据意图分析、消费者调研（如果有内涵生产的内容，也可以选择其中的字段或者整体选择），建议外延设计（营销）所采用的渠道（可以建议已有的[渠道模板]）
    - 外延生产：根据外延设计，用定好的渠道生成相应内容
4. 消费者模拟：根据内容的具体形态模拟真实用户，对[内容生产]部分的每个字段给予评价。例如，如果是对话式 Chatbot，“真实用户”需要能和场景产生对话。例如，如果是学习的文本材料，“真实用户”需要能反馈学习体验。
    1. 在这里，消费者模拟记录的是每次模拟的记录，是一个表格。每次新建一个条目，意味着需要选择让“消费者”[simulator]模拟的内容类型、交互类型等，需要充分考虑一下，一定会包括对话式、全盘学习式等。这也是[后台设置界面]需要设计的，可以先预置一些情况。Simulator的画像来自[消费者调研]中用户选中的，也可以是用户自定义。
5. 评估：一套根据意图分析、消费者调研、内容类型等生成的评估体系（也可以有预置模板），对这个项目做出全盘评价；消费者模拟的评估也会被考虑进来。这里要给出几个明确的修改建议，然后用户可以选择是否采纳，采纳的话要由全局 Agent 和用户沟通是否修改。

内涵设计产出方案及字段建议，创作者可选择字段模板。

字段模板处可以设置字段及其关联关系。在内涵设计部分可以补充和重新定义具体的字段及关联关系。

字段的关联关系指的是：如果后一个字段关联了前一个字段，则后一个字段以前一个字段的输出为输入的一部份。

## 系统前端界面

分为两个主要的界面。

### 后台设置界面

后台必须包括：

- 项目设置：主要用于意图分析，也是项目所有字段的输入。
- 创作者特质（全局设计，在开始项目时先选创作者特质）：作为项目所有字段的输入。
- Agent 设置，可以包括 Skill 管理，还有一些 Tool Call 的 Tool，比如 DeepResearch 等。
- 字段模板：一些可复用的字段及其关联关系。每个字段模板可新建若干字段，每个字段可以包括名称、类型、AI生成提示词（这个部分直接传递给大模型指导该字段内容生产）、生成前提问（也会作为提示词的一部份传递给大模型，在正是生成前以弹窗形式弹出让用户填写）、依赖字段。
- Simulator 管理
- 渠道管理
- 调试日志：每一次大模型调用的系统输入和输出、时长、token数、价格

### 内容生产界面

可选择项目及其备份的版本，每次重开加载之前的项目。

界面分左中右三栏：

- 左栏：项目进度，包括[内容生产流程]中的所有步骤，其中内涵设计、内涵生产、外延设计、外延生产这四个部分的顺序可以拖拽调整。每个部分具备已完成、完成中、未开始这三种状态。
- 中栏：内容展示区，生成的内容都在这里展示，能渲染 Markdown 格式。
- 右栏：AI Agent 区，是个正常的对话框，不论左栏中栏进展到什么部分，右边都可以对话。用 LangChain 实现真正的 Agent 架构，不允许是 if/else 结构。所有已生成的字段都可以 @，实现多种意图。

### 总体说明

【字段及其关联关系】

所有的字段都是产品，需要可编辑、可保存、可删除，每个字段的输入是什么，都可以由用户设置。这体现为：

- 在后台设置界面的字段模板项目里，可以设置可复用的字段模板
- 在内容生产界面的“内涵生产”的大纲中，每个字段都可以再定义字段关联关系，包括引用的字段模板

字段的关联关系指的是：如果后一个字段关联了前一个字段，则后一个字段以前一个字段的输出为输入的一部份。这里的关系，包括串行、并行，需要提供灵活的自定义选项。简单的串并行就不说了，一些默认的规则如：

- 如果字段 B、C 都依赖字段 A，B、C 间没有关系，则只要 A 已有输出，B、C 并行输出
- 如果字段 B 依赖字段 A，字段 C 依赖字段 A、B，则需要在 A、B 并行输出后，C 以 A、B 的输出为上下文。

初步设想是，在内容生产界面，每个字段在生成前都可以选择依赖关系，如果依赖的字段没有内容，则该字段不能生成。

【版本管理】

每个项目，只要后面的字段已经生成、需要修改前面的字段，就需要有新版本的管理。每次开新版本，需要用户输入新版本说明。

【Agent 的自主权】

每个项目都需要能够设置 Agent 自主权，也就是 Agent 不需要人检查自己推进的板块。换句话说，需要在“一键生成”前设置每个流程的回顾点。用户可以全部交给AI生成然后只看报告再跟 Agent 交流修改，也可以选择每生成一个字段就介入审核一次，这是需要设计在流程里的。

应该有一些明确的 Tool、Skill 可供 Agent 调用，比如 DeepResearch 是[消费者调研]自动调用的，其他的 Skill 等需要的时候调用。
